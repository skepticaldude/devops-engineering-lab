# Asking Good Technical Questions

## CATB Guide

Website:
- http://catb.org

Famous article:
- “How To Ask Questions The Smart Way”

### Purpose
Teaches how to:
- Ask clear technical questions
- Provide useful details
- Avoid wasting people's time
- Troubleshoot before asking

### Main Idea
Do not ask vague questions like:
```text
Linux broken pls help
```

Instead:
- Explain the problem
- Show commands used
- Include error messages
- Mention what you've already tried

# RFCs (Request for Comments)

## What RFCs Are
RFCs are official technical documents describing:
- Internet standards
- Protocols
- Networking behavior
- Communication rules

Published mainly by:
- IETF (Internet Engineering Task Force)

---

# Why RFCs Matter

RFCs are considered:
```text
"The truth"
```

Because they contain:
- Original specifications
- Official protocol behavior
- Design documents of the internet

### Important For Learning
A sysadmin should eventually learn:
- TCP
- IP
- UDP
- Email protocols
- DNS
- HTTP

Reading RFCs helps understand:
- HOW the internet works internally
- WHY systems behave the way they do

---

# RFC Learning Advice

## Key Idea
RFCs can look difficult because of:
- Formal language
- Technical wording
- Dense structure

But reading them helps develop:
- Deep understanding
- Real networking knowledge
- Troubleshooting ability

# Man Pages

## What Man Pages Are
Manual pages built into Linux systems.

Used for:
- Command documentation
- Syntax reference
- Options and flags
- System calls
- Configuration references

Example:
```bash
man ls
```
